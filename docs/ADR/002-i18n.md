# ADR-002: Language & Content Architecture

**Status:** Accepted  
**Date:** 2026-06-08

## Decision

- **Codebase:** English (identifiers, comments, commits, dev docs)
- **User-facing app:** Dutch only for v1 (`nl-NL`)
- **No runtime i18n framework** for v1 — single locale reduces complexity

## Content structure

```
src/content/nl/
├── common.json          # Shared UI: buttons, nav, errors
├── crisis.json          # Safety disclaimers, crisis resources
├── intake.json          # Onboarding intake copy
├── modules/
│   ├── 0-onboarding.json
│   ├── 1-recognition.json
│   └── ...
└── exercises/
    ├── emergency-grounding.json
    └── ...
```

## Rules for developers

1. Never put Dutch strings directly in TSX — import from content files
2. Use English keys: `content.modules.recognition.title` → displays Dutch value
3. Module IDs in code are English; `titleNl` or nested Dutch values in JSON
4. Placeholder content uses `[PLACEHOLDER]` marker — grep-friendly

## Future

If English locale is needed later, add `src/content/en/` and introduce `next-intl` or similar. Do not pre-build i18n infrastructure in v1.
